Apr 14, 2020 | react vue

背景

虽然Vue框架和React框架都提倡开发声明式组件，但是有时对于可复用的弹出式对话框(Modal)组件，在多处不同业务组件中都有被用到，需要重复声明，会有点累赘的感觉。

React框架下创建命令式组件

关键Api

ReactDOM.render(element, container[, callback])

在提供的container里渲染一个React元素，并返回对该组件的引用（或者针对无状态组件返回null）。
如果React元素之前已经在container里渲染过，这将会对其执行更新操作，并仅会在必要时改变DOM以映射最新的React元素。参考

ReactDOM.unmountComponentAtNode(container)

从DOM中卸载组件，会将其事件处理器（event handlers）和state一并清除。如果指定容器上没有对应已挂载的组件，这个函数什么也不会做。如果组件被移除将会返回true，如果没有组件可被移除将会返回false。

命令式组件优点和缺点

优点

1、调用方便
2、减少不必要的初次渲染，提高页面初次渲染效率
3、不用管理组件的销毁，或者重置组件内部状态

缺点

1、不符合Vue框架和React框架，提倡开发声明式组件的思想
2、因React事件冒泡机制，父组件捕获不到命令式子组件的事件(不过一般也不需要)
